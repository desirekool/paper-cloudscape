# Session Protocol — 2026-07-12

## Summary
Fixed section detection timing bug where section name lagged behind visible content when scrolling left-to-right. Added current-section highlighting in progress labels.

## Bug
Section name in HUD stats card updated at `scrollTarget` (center of section cloud group), but clouds became visible ~7% earlier. Result: user saw BERUFSERFAHRUNG clouds while HUD still showed "KERNKOMPETENZEN", and by the time the name updated, clouds were already partially off-screen.

## Root Cause
`sky.ts:27` used `[...cvPhases].reverse().find(p => progress >= p.scrollTarget)` — section changed only at exact center of content zone. Content spans ±12 units from center, becomes visible ~progress 0.291 for BERUFSERFAHRUNG, but name didn't change until progress 0.36.

## Fixes
1. **`src/systems/sky.ts`** — Changed to closest-scrollTarget (midpoint-threshold) detection so section name updates near when content first enters view
2. **`src/style.css`** — Added `.active` class highlighting for current section in progress-labels, `text-overflow: ellipsis` on `.time-name`, overflow protection on `.progress-labels span`
3. **`src/scene/clouds.ts`** — Gap-based redistribution: decorative clouds within 32u of any CV phase center get pushed to safe zones between phases (weighted by gap size), keeping all 51 clouds but clear of text
4. **`index.html`** + **`src/style.css`** — Progress labels repositioned from flex `space-between` to absolute `left` at actual scrollTarget values (0%, 12%, 24%, 36%, 50%, 62%, 74%)
